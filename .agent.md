# Agent 提示

这个文档是用来给 Agent （你）提供提示的，包含程序结构方案和一些 LLM 没有学到的知识。

## 程序简介

2048游戏经典规则

## Dart语言变更

### Dot shorthands

Dart 3.10 引入了 dot shorthands 的语法糖，允许在类型明确的情况下省略类名，直接以点开头，例如

```dart
// 传统写法
MyEnum stat = MyEnum.foo;
Int id = Int.parse('123');

// dot shorthands 写法
MyEnum stat = .foo;
Int id = .parse('123');
```

特别地，类主构造函数可以写为`.new()`，例如`Uuid uuid = .new();`，但是你（Agent）不应使用。

## 程序构建指导

使用 $UI = f(State)$ 方式构建。UI 层主要依赖 BoardProvider 的 stackSorted 状态构建，不直接操作borad。动画完全使用隐式动画。棋盘上的块使用Stack布局。

### 关于动画

假如移动动画的时长为200ms，合并动画的时长为100ms，那么每个块Widget应该开一个300ms的动画，前200ms大小为0（Sized.shrink），200~
300ms大小逐渐从0变成1，之后就不用缩放了。注意使用id区分不同的块。

### 关于颜色和主题

创建一个ThemeExtension，包含2048游戏相关颜色。提供 classic 构造函数，包含经典配色。
